#+begin_example
  鉴于部分用户希望使用 root 账户登录桌面，故撰写本节

  root 账户拥有系统最高权限，不当使用 root 账户可能导致系统损坏，因此以其登录图形界面存在极高的安全风险

  以下操作应谨慎执行，风险自负
#+end_example
* 文件结构
#+begin_example
  /usr/local/etc/
  └── pam.d/
      ├── gdm-password # GDM 登录 PAM 配置
      ├── lightdm      # LightDM 登录 PAM 配置
      └── sddm         # SDDM 登录 PAM 配置
#+end_example
* GDM
#+begin_example
GDM，即 GNOME Display Manager，GNOME 显示管理器
#+end_example

打开 */usr/local/etc/pam.d/gdm-password* 文件，注释掉 _account requisite pam_securetty.so_ 这一行（在行首添加 #）

#+begin_example
#account requisite pam_securetty.so
#+end_example

重启 GDM 服务：

#+begin_src sh 
  $ service gdm restart
#+end_src
* LightDM
#+begin_example
LightDM，即 Light Display Manager，轻量级显示管理器
#+end_example

编辑 */usr/local/etc/pam.d/lightdm* 文件，注释掉 _account requisite pam_securetty.so_ 这一行（在行首添加 #）。

#+begin_example
#account requisite pam_securetty.so
#+end_example
重启 LightDM 服务：

#+begin_src sh 
  $ service lightdm restart
#+end_src
* SDDM
#+begin_example
SDDM，即 Simple Desktop Display Manager，简易桌面显示管理器
#+end_example

更改 */usr/local/etc/pam.d/sddm* 文件：将 include 之后的 login，改为 system，共计四处

#+begin_src sh 
  $ service sddm restart
#+end_src

重启 SDDM 服务
