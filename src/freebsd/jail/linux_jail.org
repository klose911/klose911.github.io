Linux Jail 是 FreeBSD 操作系统中的一项功能，用于在 Jail 中运行 Linux 二进制文件和应用程序。其原理是在 FreeBSD 内核中集成兼容层，将 _Linux 系统调用_ 转换为对应的 _FreeBSD 原生系统调用_ 

#+begin_example
  Linux Jail 旨在无需单独部署 Linux 虚拟机或运行环境的情况下，使 FreeBSD 系统能够运行 Linux 软件。

  以下介绍在 FreeBSD Jail 中部署 Linux Jail 的初始化配置
#+end_example
* 准备工作
#+begin_example
  本节将所有 Jail 绑定到虚拟网络接口 lo1，使其构成 FreeBSD 系统下的一个局域网，其中 FreeBSD 主机充当该局域网的网关
#+end_example

所有 Jail 的网络流量必须通过网络接口 lo1，因此需要启用网络转发

#+begin_example
  本节使用 pf 防火墙实现此功能

  需要配置 pf 防火墙以实现网络访问控制
#+end_example

** 准备网络接口
添加克隆网络接口 lo1 并启用：
#+begin_src sh 
  $ sysrc cloned_interfaces+="lo1"
  $ service netif cloneup
#+end_src

** 准备 pf 防火墙
提供两种配置方式，可根据需要选用

*** 方案一
pf 防火墙的 *表* _table_ 是一种命名结构，用于保存地址和网络的集合，表中的地址可通过 NAT 访问网络

#+begin_example
  即使没有相关规则引用某表，persist 标志也使防火墙始终保留该表，从而防止防火墙规则重新加载时自动清空该表
#+end_example

编辑 */etc/pf.conf* 文件，加入以下配置：

#+begin_src sh 
  table <jails> persist
  pass out on em0 inet from <jails> to any nat-to (em0)
#+end_src

#+begin_example
  实际使用时请将接口名 em0 以及各 Jail 的 IP 地址替换为自己环境中的实际配置
#+end_example

可使用 pfctl 对 jails 表添加或删除条目，以控制网络访问。例如：
+ pfctl -t jails -T add 192.168.5.1 将 192.168.5.1 加入 jails 表使其可以访问网络
+ pfctl -t jails -T delete 192.168.5.1 将 192.168.5.1 移出 jails 表使其无法访问网络

#+begin_example
此方法需手动管理，但灵活性更高
#+end_example

*** 方案二
直接在 /etc/pf.conf 文件中写下规则：

#+begin_src sh 
  pass out on em0 inet from 192.168.5.1 to any nat-to (em0)
#+end_src

#+begin_example
  此方法允许 192.168.5.1 访问网络，其特点是规则固化在配置文件中，对于无特殊需求的场景较为便捷
#+end_example

** 管理 Jail 配置文件
在 */etc/jail.conf* 文件中，加入以下内容（如果没有则新建）。关键配置项包括：
+ devfs_ruleset 定义 devfs 的规则集
+ enforce_statfs 控制 Jail 中挂载点的可见性，取值为 0（无限制）、1（仅根目录下可见）或 2（默认，仅根目录所在挂载点可操作）

#+begin_src sh 
  debian {                               # Jail 名称
      host.hostname = "debian";             # 设置 Jail 的主机名
      mount.fstab = "/etc/fstab.debian";    # Jail 使用的 fstab 文件：启动或关闭 Jail 时，挂载或卸载对应的文件系统
      path = "/usr/jails/debian";           # Jail 的根目录路径
      devfs_ruleset = 4;                     # Jail 挂载 devfs 的规则集，0 表示无规则集，Jail 会继承上级规则集；
      # 仅在启用 allow.mount 和 allow.mount.devfs 且 enforce_statfs 小于 2 时可挂载 devfs
      enforce_statfs = 1;                    # 设置为 0：所有挂载点可用，无限制
      # 设置为 1：仅 Jail 根目录下的挂载点可见，且路径前缀中的 Jail 根目录部分会被剥离（例如 /usr/jails/debian/mnt 在 Jail 内显示为 /mnt）
      # 设置为 2（默认）：只能在 Jail 根目录所在挂载点操作，无法挂载 devfs、tmpfs 等
      allow.mount;                          # 允许挂载文件系统
      allow.mount.devfs;                     # 允许挂载 devfs
      exec.start = "/bin/true";              # Jail 启动时执行的命令
      exec.stop = "/bin/true";               # Jail 停止时执行的命令
      persist;                               # 允许 Jail 在无任何进程情况下仍然存在
      allow.raw_sockets;                      # 允许使用 raw Socket，例如 ping
      interface = "lo1";                      # 使用 lo1 作为网络接口
      ip4.addr = 192.168.5.1;                 # 指定 IPv4 地址
      ip6 = "disable";                        # 禁用 IPv6
  }
#+end_src

exec.start 指定启动 Jail 时运行的命令

#+begin_example
  在 FreeBSD 中创建 Jail 时，一般使用 exec.start = 'sh /etc/rc' 来调用 rc 系统启动服务

  Debian 使用 systemd 作为初始化系统，而 Jail 缺少必要的 cgroup 挂载和系统权限，无法使用 systemd，因此无法直接运行相应命令（但 service 命令仍可使用）
#+end_example

此处使用 /bin/true 安全返回 true（成功状态），不执行任何操作

#+begin_example
  例如，在 debian Jail 中启用 sshd 服务后（执行 service ssh start），重启 Jail 时 sshd 服务不会随 Jail 自动启动

  此时可设置 exec.start = 'service ssh start'，以确保启动 Jail 时自动启动 sshd 服务
#+end_example

要启用更多服务，可按如下方式编写。要启用更多服务，可按如下方式编写：
#+begin_src sh 
  exec.start += 'service ssh start'
  exec.start += 'service dbus start'
#+end_src

exec.stop 指定停止 Jail 时运行的命令
#+begin_example
FreeBSD Jail 通常使用 sh /etc/rc.shutdown

同样由于 systemd 的限制，此处使用 /bin/true 安全返回 true
#+end_example

** 管理挂载文件
创建 */etc/fstab.alpine* 文件。其中 /tmp 挂载已被注释，避免将整个宿主机 /tmp 目录暴露给 Jail，以提高安全性：

#+begin_src sh 
  devfs      /usr/jails/alpine/dev      devfs       rw                      0  0
  tmpfs      /usr/jails/alpine/dev/shm  tmpfs       rw,size=1g,mode=1777    0  0
  fdescfs    /usr/jails/alpine/dev/fd   fdescfs     rw,linrdlnk             0  0
  linprocfs  /usr/jails/alpine/proc     linprocfs   rw                      0  0
  linsysfs   /usr/jails/alpine/sys      linsysfs    rw                      0  0
  #/tmp       /usr/jails/alpine/tmp      nullfs      rw                      0  0  # 注释掉可避免将整个宿主机 /tmp 目录暴露给 Jail
#+end_src

** 管理 Jail 模板
在 */etc/jail.conf* 文件中写入：

#+begin_src sh 
  alpine {                               # Jail 名称
      host.hostname = "alpine";             # 设置 Jail 的主机名
      mount.fstab = "/etc/fstab.alpine";    # Jail 使用的 fstab 文件
      path = "/usr/jails/alpine";           # Jail 根目录路径
      devfs_ruleset = 4;                     # devfs 挂载规则集
      enforce_statfs = 1;                    # 设置挂载点可见性
      allow.mount;                          # 允许挂载文件系统
      allow.mount.devfs;                     # 允许挂载 devfs
      exec.start = "/bin/true";              # minirootfs 中未初始化系统，暂时使用 /bin/true，后续会设置 openrc
      exec.stop = "/bin/true";               # 停止时使用 /bin/true
      persist;                               # 即使无进程也保持 Jail 存活
      allow.raw_sockets;                      # 允许使用 raw Socket
      interface = "lo1";                      # 指定网络接口
      ip4.addr = 192.168.5.4;                 # 分配 IPv4 地址
      ip6 = "disable";                        # 禁用 IPv6
  }
#+end_src

** 为基本系统配置 OpenRC

#+begin_src sh 
  freebsd $ jexec alpine /bin/sh        # 初始仅有 sh，请留意 Shell 提示符的变化

  alpine # touch /run/openrc/softlevel      # 在 Docker、Chroot 或 Jail 环境中使用 OpenRC 时建议创建此文件
  alpine # exit    # 留意 Shell 提示符的变化

#+begin_src sh 
  $ pkg install xephyr
#+end_src

在 FreeBSD 中启用：
#+begin_src sh 
  $ Xephyr :1 -listen tcp
#+end_src

参数说明：
+ :1 即 DISPLAY 值中的 displaynumber
  #+begin_example
    FreeBSD 系统 IP 为 10.0.2.15，故完整的 DISPLAY 值为 10.0.2.15:1.0

    因为 FreeBSD 系统 X Server 的 displaynumber 值为 0，故从 1 开始
  #+end_example
+ -listen tcp 监听 TCP 端口

在 Jail 中，使用与前述相同的方法：
#+begin_src sh 
  $ export DISPLAY=10.0.2.15:1.0
  $ xclock&
#+end_src

#+begin_example
  四个 Jail 可以同时在 FreeBSD 开启的一个 Xnest/Xephyr 窗口中打开 xclock

  但此时没有窗口管理器，xclock 缺少窗口装饰和基本交互功能
#+end_example
可在执行 xclock 前先执行 jwm，如下：
#+begin_src sh 
  $ export DISPLAY=10.0.2.15:1.0
  $ jwm &
  $ xclock&
#+end_src

#+begin_example
  jwm 执行一次即可，无需在每个 Jail 中分别执行

  此处使用 jwm 是因为其轻量级，Xfce 等桌面环境也可使用，可根据需要选用
#+end_example

** 共享主机 socket 方式
先在 FreeBSD 系统上执行：
#+begin_src sh 
  $ xhost +
#+end_src

#+begin_example
  xhost + 将关闭访问控制，存在安全风险

  更安全的方式是指定允许访问的 IP 地址
#+end_example

随后在 Jail 的 fstab 文件中加入下面内容，以 ubuntu Jail 的 fstab 为例，其他 Jail 参照修改即可：
#+begin_src sh 
  /tmp/.X11-unix   /usr/jails/ubuntu/tmp/.X11-unix    nullfs   ro   0  0
#+end_src

#+begin_example
必要时先 mkdir -p /usr/jails/ubuntu/tmp/.X11-unix，确保有挂载点
#+end_example

重启 jail 后，在 jail 中执行：
#+begin_src sh 
  $ export DISPLAY=:0.0
  $ xclock &
#+end_src

前文提到 fstab 文件中有下面这样一行：
#+begin_src sh 
  #/tmp   /usr/jails/ubuntu/tmp   nullfs  rw    0  0
#+end_src
#+begin_example
  该写法将 FreeBSD 的 /tmp 目录暴露给 Jail 并且可读写，破坏隔离性，因此予以注释
#+end_example

而仅挂载 /tmp/.X11-unix 可显著提高安全性

** X Server TCP 监听加 xhost 连接管理
#+begin_example
此处使用 SDDM 桌面管理器，如果使用其他桌面管理器，请参考相应文档，原理相同
#+end_example

在 FreeBSD 内编辑或新建 */usr/local/etc/sddm.conf* 文件，修改 ServerArguments 内容如下：
#+begin_src sh 
  [X11]
  ServerArguments=-listen tcp
#+end_src

重启后，在 FreeBSD 上用以下方式加入 Jail IP 以允许访问：
#+begin_src sh 
  $ xhost + 192.168.5.1
#+end_src

#+begin_example
这种方式比 xhost + 更安全，仅允许指定 IP 访问
#+end_example

随后在 Jail 中，执行：
#+begin_src sh 
  $ export DISPLAY=:0.0
  $ xclock &
#+end_src

#+begin_example
  将 DISPLAY 设为 :0.0、127.0.0.1:0.0、10.0.2.15:0.0 均可正常使用，上述几种方法均可尝试
#+end_example

** 附录与补遗
+ TCP 监听默认关闭的原因之一是宿主机 X Server 通过 TCP 监听配合 xhost 连接管理的方式安全性较差
+ 共享宿主机 Socket 的方式，需注意仅挂载 /tmp/.X11-unix 而非整个 /tmp，并使用 xhost +指定 IP 而非 xhost +
  #+begin_example
    满足这两点即可保障安全性
  #+end_example
+ 带 X Server 的终端、共享主机 Socket、VNC 这三种方式值得推荐
  #+begin_example
    无论采用哪种方式，Linux Jail 中的兼容性均存在一定限制，并非所有 X 应用都能运行

    相比之下，FreeBSD Jail 的兼容性则更好
  #+end_example

#+begin_example
除共享宿主机 Socket 的方法外，其余方法同样可在非 Jail 环境中使用
#+end_example
