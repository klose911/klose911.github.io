* 快速指南

** 开始前
确保您拥有：
+ 打开的终端或命令提示符
  + 如果您之前从未使用过终端，请查看终端指南
+ 一个可以使用的代码项目
+ 一个 Claude 订阅（Pro、Max、Team 或 Enterprise）、Claude Console 账户，或通过支持的云提供商的访问权限

#+begin_example
  本指南涵盖终端 CLI

  Claude Code 也可在网页、桌面应用、VS Code 和 JetBrains IDE、Slack 中使用，以及通过 GitHub Actions 和 GitLab 进行 CI/CD
#+end_example
** 步骤 1：安装 Claude Code
+ macOS, Linux, WSL
  #+begin_src sh 
    curl -fsSL https://claude.ai/install.sh | bash
  #+end_src
+ Windows PowerShell
  #+begin_src sh 
    irm https://claude.ai/install.ps1 | iex
  #+end_src
+ Windows CMD
  #+begin_src sh 
    curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
  #+end_src

#+begin_example
  Git for Windows is recommended on native Windows so Claude Code can use the Bash tool

  If Git for Windows is not installed, Claude Code uses PowerShell as the shell tool instead

  WSL setups do not need Git for Windows
#+end_example
** 步骤 2：登录您的账户
Claude Code 需要账户才能使用。使用 claude 命令启动交互式会话，首次使用时系统会提示您登录：
#+begin_src sh 
  claude 
#+end_src

对于 Claude 订阅或 Console 账户，请按照提示在浏览器中完成身份验证。要稍后切换账户或重新身份验证，请在运行的会话中输入:

#+begin_src sh 
  /login
#+end_src

您可以使用以下任何账户类型登录：
+ Claude Pro、Max、Team 或 Enterprise（推荐）
+ Claude Console（具有预付费额度的 API 访问）
  + 首次登录时，Console 中会自动为集中成本跟踪创建一个”Claude Code”工作区
+ Amazon Bedrock、Google Vertex AI 或 Microsoft Foundry（企业云提供商）

登录后，您的凭证将被存储，您无需再次登录
** 步骤 3：启动您的第一个会话
在任何项目目录中打开您的终端并启动 Claude Code：
#+begin_src sh 
  cd /path/to/your/project
  claude
#+end_src

将看到 Claude Code 提示符，其中显示版本、当前模型和上方显示的工作目录。输入 _/help_ 查看可用命令，或输入 _/resume_ 继续之前的对话

#+begin_example
  登录后（步骤 2），您的凭证将存储在您的系统上。在凭证管理中了解更多信息
#+end_example
** 步骤 4：提出您的第一个问题
让我们从理解您的代码库开始。尝试以下命令之一：
#+begin_src sh 
  what does this project do?
#+end_src

Claude 将分析您的文件并提供摘要。您也可以提出更具体的问题：
#+begin_src sh 
  what technologies does this project use?

  where is the main entry point?

  explain the folder structure
#+end_src

您也可以询问 Claude 关于其自身功能的问题：
#+begin_src sh 
  what can Claude Code do?

  how do I create custom skills in Claude Code?

  can Claude Code work with Docker?
#+end_src

#+begin_example
Claude Code 根据需要读取您的项目文件。您不必手动添加上下文
#+end_example
** 步骤 5：进行您的第一次代码更改
现在让我们让 Claude Code 进行一些实际的编码。尝试一个简单的任务：
#+begin_src sh 
  在主文件中添加一个 hello world 函数
#+end_src

Claude Code 将：
1. 找到适当的文件
2. 向您显示建议的更改
3. 请求您的批准
4. 进行编辑

#+begin_example
Claude Code 在修改文件前始终请求许可。您可以批准单个更改或为会话启用”全部接受”模式
#+end_example
** 步骤 6：在 Claude Code 中使用 Git
Claude Code 使 Git 操作变得对话式：
#+begin_src sh 
  我更改了哪些文件？

  用描述性消息提交我的更改
#+end_src

您也可以提示更复杂的 Git 操作：
#+begin_src sh 
  创建一个名为 feature/quickstart 的新分支

  显示我最后的 5 次提交

  帮我解决合并冲突
#+end_src
** 步骤 7：修复错误或添加功能
#+begin_example
Claude 擅长调试和功能实现
#+end_example
用自然语言描述您想要的内容：
#+begin_src sh 
  向用户注册表单添加输入验证
#+end_src

或修复现有问题：
#+begin_src sh 
  有一个错误，用户可以提交空表单 - 修复它
#+end_src

Claude Code 将：
+ 定位相关代码
+ 理解上下文
+ 实现解决方案
+ 如果可用，运行测试
** 步骤 8：尝试其他常见工作流
有多种方式可以与 Claude 一起工作：
+ 重构代码
  #+begin_src sh 
    refactor the authentication module to use async/await instead of callbacks
  #+end_src
+ 编写测试
  #+begin_src sh 
    write unit tests for the calculator functions
  #+end_src
+ 更新文档
  #+begin_src sh 
    review my changes and suggest improvements
  #+end_src
+ 代码审查
  #+begin_src sh 
    review my changes and suggest improvements
  #+end_src
#+begin_example
  像与有帮助的同事交谈一样与 Claude 交谈。描述您想要实现的目标，它将帮助您实现
#+end_example
* 基本命令
以下是日常使用中最重要的命令
+ Shell 命令从您的终端运行以启动或恢复 Claude Code
  #+CAPTION: Shell 命令
  #+ATTR_HTML: :border 1 :rules all :frame boader
  | 命令              | 功能                       | 示例                              |
  | claude            | 启动交互模式               | claude                            |
  | claude "task"     | 运行一次性任务             | claude "fix the build error"      |
  | claude -p "query" | 运行一次性查询，然后退出   | claude -p "explain this function" |
  | claude -c         | 在当前目录中继续最近的对话 | claude -c                         |
  | claude -r         | 恢复之前的对话             | claude -r                         |
+ 会话命令在 Claude Code 启动后在其内部运行
  #+CAPTION: 会话命令
  #+ATTR_HTML: :border 1 :rules all :frame boader
  | 命令            | 功能             | 示例   |
  | /clear          | 清除对话历史     | /clear |
  | /help           | 显示可用命令     | /help  |
  | /exit 或 Ctrl+D | 退出 Claude Code | /exit  |

#+begin_example
  有关完整的 shell 命令列表，请参阅 CLI 参考 https://code.claude.com/docs/zh-CN/cli-reference

  有关完整的会话命令列表，请参阅 命令参考 https://code.claude.com/docs/zh-CN/commands
#+end_example
* 初学者专业提示
有关更多信息，请参阅[[https://code.claude.com/docs/zh-CN/best-practices][最佳实践]] 和[[https://code.claude.com/docs/zh-CN/common-workflows][常见工作流]] 
+ 对您的请求要具体
  #+begin_example
    不要说：“修复错误”

    尝试：“修复登录错误，用户输入错误凭证后看到空白屏幕”
  #+end_example
+ 使用分步说明，将复杂任务分解为步骤：
  1. 为用户配置文件创建新的数据库表
  2. 创建 API 端点以获取和更新用户配置文件
  3. 构建允许用户查看和编辑其信息的网页
+ 让 Claude 先探索：
  #+begin_example
    在进行更改之前，让 Claude 理解您的代码：

    分析数据库架构

    构建一个仪表板，显示英国客户最常退货的产品
  #+end_example
+ 使用快捷方式节省时间
  + 输入 / 查看所有命令和 skills
  + 使用 Tab 进行命令补全
  + 按 ↑ 查看命令历史
  + 按 Shift+Tab 循环切换权限模式
